# 中文论文题名设计

## 目标

为每篇论文保存独立的中文题名，并在现有研究工作流中显示它。英文 `title`
始终是论文的原始题名和稳定标识；中文题名由大模型按需生成，也可以人工修订。

## 范围

- 新增可空的 `title_zh` 字段，并让已有数据库平滑升级。
- 在管理页提供批量补全入口，只处理 `title_zh` 为空的论文。
- 管理页允许单篇编辑和清空中文题名。
- 库存列表、阅读侧栏、阅读页、复习卡片统一显示英文主行和中文副标题。
- 搜索同时匹配英文题名与中文题名。

本次不翻译摘要、讲解、笔记或全文，不更改英文题名，不自动为新收录论文调用模型。

## 数据模型

`papers.title_zh` 为可空文本列。所有列表和单篇查询都返回该字段；手动添加、
编辑、采集、PDF 导入和 MCP 更新操作都必须保留已有的中文题名，除非请求明确
更新或清空它。

数据库初始化需要具备幂等迁移能力：新数据库直接含该列；已有数据库启动后补列，
不得重建数据表或丢失任何论文数据。

## 翻译工作流

管理页显示“待翻译 N 篇”，其中 N 是 `title_zh` 为空且英文题名非空的论文数。
用户点击批量操作后，服务端逐篇调用当前已配置的 OpenAI 兼容大模型，并在每篇完成后
立即保存结果。用户可以停止任务；已保存的结果保留。

模型提示要求：只返回一行忠实、简洁、符合中文学术表达的题名；保留必要的缩写、
模型名、数据集名和专有名词；不添加说明、引号、Markdown 或额外文本。服务端会清理
首尾空白和常见包裹符号，并拒绝空值或多段说明。

批量任务只更新开始时仍为空的 `title_zh`。人工填写过的中文题名永不被覆盖。单篇
失败、超时、模型未配置或返回无效内容时，英文题名保持可用，失败记录进入任务摘要，
其他论文继续处理。

## API 与界面

新增用于读取待翻译数量、启动批量任务和更新单篇 `title_zh` 的 API。批量接口采用
现有 NDJSON 进度协议，提供总数、当前论文、成功、失败、停止和最终汇总事件。

题名渲染抽为前端共享辅助函数：英文题名是主行，存在 `title_zh` 时在下方显示为较轻的
中文副标题。管理页编辑入口将中文题名与其他元数据一并提交；清空该字段后，论文会在
下一次批量翻译中重新成为候选。

## 验收与测试

- 旧数据库升级后保留原论文，且新列可读写。
- 批量任务仅翻译空值，重复运行不覆盖已有或人工编辑的翻译。
- 未配置模型、超时、空响应和单篇异常均能返回可理解的汇总，且不会中断其余任务。
- 题名在管理、阅读、复习和库内列表都以英文主行、中文副标题显示。
- 搜索能命中英文题名和中文题名。
- 现有 Node/Python 测试继续通过，并为新增数据库和接口行为补充针对性测试。
